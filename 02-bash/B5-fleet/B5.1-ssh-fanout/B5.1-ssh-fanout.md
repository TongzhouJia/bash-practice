# KP B5.1 — SSH Fan-out (run commands across many hosts)
**looping `ssh` over hosts · collecting output · `[host]` prefixes · continuing past errors · per-host exit codes · reading a host list**

**10 tasks · ⭐⭐ · RHEL 10 · server1→server2/repo · needs real hosts; the loop skeleton can be checked anywhere**

> Sample: `samples/hosts.txt` (one host per line, `#` for comments). Running the same command against a batch of machines is routine. The core is **key-based ssh + a loop + not aborting on error + output tagged with the hostname**.
> ⚠️ ssh has to reach real hosts (set up keys first, see task 7); run these from server1 against server2/repo.

---

**1.** Loop over `hosts.txt`, skipping comment (`#`) and blank lines.
*Acceptance:* `grep -vE '^\s*#|^\s*$' hosts.txt | while read -r h; do …; done` → visits `server1 server2 repo`.

**2.** Run `uptime` on each host via `ssh`.
*Acceptance:* `for h in $(...); do ssh "$h" uptime; done` (real hosts).

**3.** **Continue on error**: if one host is unreachable, keep going to the rest.
*Acceptance:* a down host prints `UNREACHABLE` but the loop still processes the others.

**4.** Prefix each host's output with `[host]` so a merged stream is readable.
*Acceptance:* `ssh "$h" 'df -P /' | sed "s/^/[$h] /"` → lines like `[server2] /dev/… 55% /`.

**5.** Use safe ssh options for scripts: `BatchMode=yes` (never prompt) and `ConnectTimeout=5`.
*Acceptance:* `ssh -o BatchMode=yes -o ConnectTimeout=5 "$h" true` — hangs/prompts are avoided.

**6.** Run a **local** script on a remote host without copying it first.
*Acceptance:* `ssh "$h" 'bash -s' < local.sh` → the script runs remotely.

**7.** Set up **key-based** auth so the loop doesn't prompt for passwords.
*Acceptance:* `ssh-keygen` then `ssh-copy-id user@server2` → subsequent `ssh server2` needs no password.

**8.** Count how many hosts are **reachable** (collect per-host success).
*Acceptance:* tally `ok`/`fail` across the loop → e.g. `reachable=2 / 3`.

**9.** Read the host list from a file and run an **arbitrary** command passed as arguments.
*Acceptance:* the command after the hostfile is what runs on each host.

**10.** Write `fanout.sh <hostfile> <cmd...>` that runs `<cmd>` on every host, prefixes output with `[host]`, continues on failure, and exits non-zero if any host failed.
*Acceptance:* `./fanout.sh hosts.txt uptime` → per-host prefixed output; exit `1` if any host failed.
