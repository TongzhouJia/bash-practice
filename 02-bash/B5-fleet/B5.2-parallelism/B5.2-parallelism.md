# KP B5.2 — Parallelism
**background `&` + `wait` · `xargs -P` for parallelism · capping concurrency · per-job exit codes · serial vs parallel speedup**

**8 tasks · ⭐⭐ · RHEL 10 · server1 · run each and match the result**

> Running a batch serially is slow — ssh-ing into 100 machines one at a time takes all day. Parallelism is the main speedup: `&`+`wait` for manual concurrency, `xargs -P N` to cap it.

---

**1.** Launch 3 jobs in the background with `&` and `wait` for all to finish.
*Acceptance:* all 3 `job N done` print (order varies), then `all done`.

**2.** Time **serial vs parallel** for three `sleep 1`s.
*Acceptance:* serial ≈ `3s`; parallel (`sleep 1 &` ×3 then `wait`) ≈ `1s`.

**3.** Use `xargs -P4` to process a list of inputs in parallel.
*Acceptance:* `printf '%s\n' alpha bravo charlie delta | xargs -P4 -I{} sh -c 'echo processed {}'` → all four processed.

**4.** Use the `-I{}` placeholder to put each input into the command.
*Acceptance:* `{}` is replaced by each input value.

**5.** Cap concurrency at 2 with `xargs -P2` (observe only 2 run at once).
*Acceptance:* `seq 1 4 | xargs -P2 -I{} sh -c 'echo start {}; sleep 0.2; echo end {}'` → `start 1`/`start 2` before any `start 3`.

**6.** Capture the **exit status** of a specific background job with `wait <pid>`.
*Acceptance:* `( exit 3 ) & pid=$!; wait "$pid"; echo $?` → `3`.

**7.** Parallelize a loop but **cap** at 2 concurrent jobs (drain with `wait` every 2).
*Acceptance:* all 5 jobs complete, never more than 2 running at once.

**8.** Process several files in parallel with `xargs -P4` (e.g. line-count each).
*Acceptance:* `ls *.log *.csv | xargs -P4 -I{} sh -c 'wc -l < {}'` → counts for all files.
