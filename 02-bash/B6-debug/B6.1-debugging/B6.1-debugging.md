# KP B6.1 — Debugging
**`bash -x` · `set -x`/`+x` · `PS4` · `bash -n` syntax checks · `set -v` · `declare -p` · `set -u` to catch typos**

**8 tasks · ⭐ · RHEL 10 · server1 · run each and match the result**

> When a script misbehaves, do not guess — **`bash -x` prints every command after expansion**, so you see exactly what a variable turned into. This section is the biggest time-saver in the set.

---

**1.** Trace a whole script with `bash -x` (script sets `x=5`, `y=x*2`, echoes `y`).
*Acceptance:* trace shows `+ x=5`, `+ y=10`, `+ echo 10`, then `10`.

**2.** Trace only a region with `set -x` … `set +x`.
*Acceptance:* the traced line (`+ echo hi`) shows with a `+`; lines outside aren't traced.

**3.** Customize `PS4` so the trace shows line numbers (`PS4='+${LINENO}: '`).
*Acceptance:* traced lines are prefixed with the line number (e.g. `+line 7: …`). In a real multi-line script each command shows its own line.

**4.** Check a script for **syntax errors without running it** using `bash -n` (script has a `for` missing its `done`).
*Acceptance:* `bash -n bad.sh` reports `syntax error: unexpected end of file`, exit non-zero — and nothing actually executed.

**5.** Use `set -v` (or `bash -v`) to echo each line **as read** before running it.
*Acceptance:* input `echo one / echo two` prints each command line then its output.

**6.** Inspect a variable/array's real contents with `declare -p`.
*Acceptance:* `arr=(a b c); declare -p arr` → `declare -a arr=([0]="a" [1]="b" [2]="c")`.

**7.** Catch a **mistyped variable name** with `set -u`.
*Acceptance:* `set -u; name="Tongzhou"; echo "$naem"` → `naem: unbound variable`, exit non-zero (instead of silently printing empty).

**8.** Trace a **function's** internals with `bash -x`.
*Acceptance:* a `double(){ echo $(($1*2)); }` called as `double 21` traces `+ double 21`, `+ echo 42`, then `42`.
