# KP B3.3 — Loops & Reading Line-by-Line
**`for` over lists/C-style/`{a..b}` · `while`/`until` · `while IFS= read -r` · the pipeline-subshell trap · looping over globs · `break`/`continue`**

**10 tasks · ⭐⭐ · RHEL 10 · server1 · run each in bash and match the result**

> Samples are in `samples/`. Processing a file line by line is daily work. **The canonical form: `while IFS= read -r line; do …; done < file`** — remember `IFS=` (preserves whitespace), `-r` (no backslash escapes), and `< file` (do not use `cat | while`, see task 8).

---

**1.** `for` over a list: print `Creating <x>` for `alpha bravo charlie`.
*Acceptance:* three `Creating …` lines.

**2.** C-style `for` printing `1 2 3 4 5` on one line.
*Acceptance:* `for ((i=1;i<=5;i++))` → `1 2 3 4 5`.

**3.** `for` over a brace range `{1..3}` printing `node1 node2 node3`.
*Acceptance:* three lines `node1`…`node3`.

**4.** `while` with a counter printing `w1 w2 w3`.
*Acceptance:* `while (( i<=3 ))` with `((i++))`.

**5.** `until` loop printing `u1 u2 u3` (loops **until** the condition becomes true).
*Acceptance:* `until (( i>3 ))`.

**6.** Read `passwd.sample` line by line, splitting on `:`, and print `user:uid` (fields 1 and 3).
*Acceptance:* `while IFS=: read -r u _ uid _; do echo "$u:$uid"; done < passwd.sample` → `root:0`, `daemon:1`, …

**7.** Show `IFS= read -r` preserves leading whitespace: feed `"  indented"` and `"plain"`, print each in brackets.
*Acceptance:* `[  indented]` (spaces kept) and `[plain]`.

**8.** Demonstrate the **pipe subshell pitfall**: count lines with `printf … | while read` (count is lost) vs `while read … done < <(printf …)` (count survives).
*Acceptance:* piped → `count=0`; process-substitution/redirect → `count=3`.

**9.** Loop over `*.txt` files in a directory safely (enable `nullglob` so an empty match doesn't loop once on the literal pattern).
*Acceptance:* with two `.txt` files → `file: f1.txt`, `file: f2.txt`.

**10.** `break`/`continue`: loop `1..5`, skip `3`, stop at `5` (print `1 2 4`).
*Acceptance:* `continue` on 3, `break` on 5 → `1`, `2`, `4`.
