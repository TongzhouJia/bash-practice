# KP B3.8 — Arithmetic & Numbers
**`(( ))` / `$(( ))` · increment · comparison/ternary · integer division & modulo · `bc` for floats · `printf` rounding · number bases · `$RANDOM`**

**8 tasks · ⭐ · RHEL 10 · server1 · run each in bash and match the result**

> Bash natively does **integer** arithmetic only (`$(( ))`). For decimals hand off to `bc` or format with `printf`. This section runs through the common moves for handling numbers.

---

**1.** Evaluate `3 + 4 * 2` with `$(( ))` (respecting precedence).
*Acceptance:* `echo $((3+4*2))` → `11`.

**2.** Increment a variable in place.
*Acceptance:* `n=5; ((n++)); echo $n` → `6`.

**3.** Ternary inside `$(( ))`: print `100` if `a>5` else `0`, for `a=7`.
*Acceptance:* `echo $(( a>5 ? 100 : 0 ))` → `100`.

**4.** Integer division and modulo of `17` by `5`.
*Acceptance:* `echo "$((17/5)) $((17%5))"` → `3 2`.

**5.** Floating-point `10/3` to 2 decimals using `bc`.
*Acceptance:* `echo "scale=2; 10/3" | bc` → `3.33`.

**6.** Round `3.14159` to 2 decimals with `printf`.
*Acceptance:* `printf "%.2f\n" 3.14159` → `3.14`.

**7.** Bases: interpret hex `FF` and binary `1010` as decimals, and print `255` back as hex.
*Acceptance:* `echo "$((16#FF)) $((2#1010))"` → `255 10`; `printf "%x\n" 255` → `ff`.

**8.** Roll a die (1–6) using `$RANDOM`.
*Acceptance:* `echo $((RANDOM%6+1))` → a number in `1..6`.
