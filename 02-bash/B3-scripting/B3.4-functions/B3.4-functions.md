# KP B3.4 — Functions
**defining and calling · arguments `$1 $@ $#` · `local` · return status `return` vs output `echo` · capturing output `$(func)` · default arguments**

**8 tasks · ⭐ · RHEL 10 · server1 · run each in bash and match the result**

> Functions make scripts reusable and readable. Two things matter: **variables are global by default, so always use `local` inside a function**; and **a function's "return value" is what it prints to stdout (capture with `$(func)`) — `return` only gives a 0–255 exit status.**

---

**1.** Define `greet` taking a name, printing `Hi, <name>`; call `greet Tongzhou`.
*Acceptance:* `Hi, Tongzhou`.

**2.** Show `local` scoping: a variable declared `local` inside a function doesn't overwrite the same-named variable outside.
*Acceptance:* inside → `in:inside`; after the call → `out:outside`.

**3.** Use a function's **return status** as a condition: `is_even` returns success when `$1` is even; test `4` and `5`.
*Acceptance:* `4 even`; `5 odd`.

**4.** Capture a function's **output**: `sum` echoes `$1+$2`; capture with `$( )` into a variable.
*Acceptance:* `sum 3 4` captured → `r=7`.

**5.** Inside a function print the argument count and all args (`$#`, `$*`).
*Acceptance:* `count_args a b c` → `n=3: a b c`.

**6.** Default parameter: `greet2` uses `local name=${1:-guest}`. Call with no arg and with `Alice`.
*Acceptance:* no arg → `Welcome, guest`; `Alice` → `Welcome, Alice`.

**7.** "Return a string" via `echo`: `maxof` prints the larger of two numbers; capture it.
*Acceptance:* `maxof 7 3` → `max=7`.

**8.** Use a function inside a loop: `double` echoes `$1*2`; call it for `1 2 3`.
*Acceptance:* `2 4 6`.
