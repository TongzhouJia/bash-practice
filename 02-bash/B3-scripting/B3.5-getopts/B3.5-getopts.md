# KP B3.5 — Option Parsing with `getopts`
**boolean flags `-v` · options with arguments `-f file` + `OPTARG` · silent error mode `:optstring` · required-flag checks · `shift $((OPTIND-1))` · bundled short flags**

**8 tasks · ⭐ · RHEL 10 · server1 · build one parser script, run it with the invocations below**

> `getopts` is the standard way to give a script a proper command-line interface like `-v -f file`. Write the **reference script `optdemo.sh`** below first, then run it with various arguments and check the output.

**Reference script:**
```bash
#!/bin/bash
set -euo pipefail
usage(){ echo "Usage: $(basename "$0") [-v] [-n name] -f file [args...]" >&2; exit 1; }
verbose=0; name="guest"; file=""
while getopts ":vn:f:" opt; do          # leading : = silent mode, handle errors yourself
  case $opt in
    v) verbose=1 ;;                       # boolean flag
    n) name=$OPTARG ;;                    # takes a value: it lands in $OPTARG
    f) file=$OPTARG ;;
    :)  echo "Option -$OPTARG requires an argument" >&2; exit 1 ;;  # missing argument
    \?) echo "Invalid option: -$OPTARG" >&2; usage ;;               # unknown option
  esac
done
shift $((OPTIND-1))                       # drop parsed options, leaving positional args
[[ -n $file ]] || { echo "Error: -f is required" >&2; exit 1; }     # required check
echo "verbose=$verbose name=$name file=$file rest=$*"
```

---

**1.** Boolean flag `-v` sets verbose. **2.** `-n name` and **3.** `-f file` capture values via `OPTARG`. Run the full form.
*Acceptance:* `./optdemo.sh -v -n Alice -f data.txt extra1 extra2` → `verbose=1 name=Alice file=data.txt rest=extra1 extra2`.

**4.** Defaults when options are omitted.
*Acceptance:* `./optdemo.sh -f only.txt` → `verbose=0 name=guest file=only.txt rest=`.

**5.** Missing option-argument is caught by the `:` case (thanks to the leading `:` in the optstring).
*Acceptance:* `./optdemo.sh -f` → `Option -f requires an argument` (stderr), exit `1`.

**6.** Invalid option is caught by the `\?` case.
*Acceptance:* `./optdemo.sh -x` → `Invalid option: -x` then the usage line, exit `1`.

**7.** Required-option check after parsing: `-f` is mandatory.
*Acceptance:* `./optdemo.sh -v` → `Error: -f is required` (stderr), exit `1`.

**8.** Combined short flags and `shift $((OPTIND-1))` for positional args.
*Acceptance:* `./optdemo.sh -vf data.txt` → `verbose=1 name=guest file=data.txt rest=` (`-vf` = `-v -f`).
