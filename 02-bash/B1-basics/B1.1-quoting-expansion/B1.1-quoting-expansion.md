# KP B1.1 — Quoting & Expansion Order
**single/double quotes · escaping · `$( )` · `$(( ))` · brace `{}` · tilde `~`**

**12 tasks · ⭐⭐⭐ · RHEL 10 · server1 as root · run each in a bash shell and match the stated result exactly**

> This is the most trap-heavy foundation in the set. Do not just reason out the answer — actually run every line and see what it prints, especially the difference between single and double quotes.

---

**1.** Print the exact literal line `Price is $5 (100% off!)` — the `$5`, the parentheses and the `%` must all appear verbatim (nothing expanded, nothing executed).
*Acceptance:* output is exactly `Price is $5 (100% off!)`.

**2.** With `name=Bash` already set, print these two lines — first with single quotes, second with double quotes:
```
single: $name
double: Bash
```
i.e. line 1 must show the literal text `$name`, line 2 must show the expanded value `Bash`.
*Acceptance:* line 1 contains the four characters `$name`; line 2 contains `Bash`.

**3.** Using command substitution `$( )`, store the short hostname (`hostname -s`) in a variable `h`, then print `Host is <h>`.
*Acceptance:* command uses `$(...)`; prints e.g. `Host is server1`.

**4.** Using arithmetic expansion `$(( ))`, print how many seconds are in one day, computed from `24*60*60` — do **not** type `86400` yourself.
*Acceptance:* prints `86400`, produced by `$((...))`.

**5.** In **one** command using brace expansion, create the four files `report-jan.log`, `report-feb.log`, `report-mar.log`, `report-apr.log` in the current directory.
*Acceptance:* exactly those 4 files exist; the command contains `{jan,feb,mar,apr}`.

**6.** In **one** command using brace expansion, create ten directories `node01 node02 … node10` (two digits, zero-padded).
*Acceptance:* `node01` … `node10` exist; the command uses `{01..10}`.

**7.** Using tilde expansion, print root's home directory — write `~root`; do **not** type `/root` literally.
*Acceptance:* prints `/root`, produced via `~root`.

**8.** A regular file named `my report.txt` (there is a space in the name) must be created, then displayed with `cat`, using quoting so the space is not treated as an argument separator.
*Acceptance:* `cat` reads the single file with no "No such file" error.

**9.** In one line, print `Today is <weekday>` where `<weekday>` comes from `date +%A`, embedded with command substitution **inside a double-quoted string**.
*Acceptance:* e.g. `Today is Friday`; the command uses `"...$(date +%A)..."`.

**10.** Inside a directory that contains `a.txt b.txt c.txt`, print the literal string `*.txt` to the screen — the glob must **not** expand into the filenames.
*Acceptance:* output is exactly `*.txt`.

**11.** You have a file `config.ini`. In **one** command using brace expansion, make a backup copy named `config.ini.bak` (use the `cp file{,.bak}` idiom).
*Acceptance:* `config.ini.bak` is an identical copy of `config.ini`; the command uses `{,.bak}`.

**12.** Run `echo "{1..3} = $(echo {1..3})"` and state its **exact** output; in your answer, explain the expansion order.
*Acceptance:* output is `{1..3} = 1 2 3` — brace expansion does **not** happen inside double quotes, but **does** happen inside the unquoted `$( )`.
