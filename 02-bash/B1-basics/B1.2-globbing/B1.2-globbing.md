> Key distinction to lock in: **globbing only matches filenames that already exist** (it is not regex), and the matching happens before the command runs, done by the shell. Create the sample directory below before starting.

**Setup (run this first to create the samples):**
```bash
mkdir -p ~/globlab && cd ~/globlab
touch app.log app.log.1 app.log.2 error.log access.log data.csv data.txt notes.md
touch .hidden.log .env
touch host1.conf host2.conf host10.conf hostname.conf
```

---

**1.** With `*`, list every file whose name ends in `.log`.
*Acceptance:* matches exactly `access.log app.log error.log` ŌĆö note `app.log.1`/`app.log.2` end in `.1`/`.2` (excluded), and the hidden `.hidden.log` is excluded too.

**2.** With the single-character wildcard `?`, list `host1.conf` and `host2.conf` **only** ŌĆö not `host10.conf`, not `hostname.conf`.
*Acceptance:* pattern is `host?.conf`; output is exactly `host1.conf host2.conf`.

**3.** Using a character **set** `[ ]`, list the conf files whose digit is `1` or `2` (`host1.conf`, `host2.conf`).
*Acceptance:* pattern uses `[12]`, e.g. `host[12].conf`.

**4.** Using a character **range** `[0-9]`, list the rotated logs `app.log.1` and `app.log.2`.
*Acceptance:* pattern is `app.log.[0-9]`; output is those two files.

**5.** Using a **negated** class `[!ŌĆ”]`, list every `.log` file whose name does **not** start with the letter `a` (so `error.log` matches; `app.log` and `access.log` do not).
*Acceptance:* pattern is `[!a]*.log`; output is exactly `error.log`.

**6.** Show the default behaviour of an **unmatched** glob, then change it with `nullglob`:
- First run `for f in *.xml; do echo "[$f]"; done` in a dir with no `.xml` files and report what prints.
- Then enable `shopt -s nullglob` and run the same loop; report what changes.
*Acceptance:* default prints the literal `[*.xml]` (loop runs once with the pattern itself); with `nullglob` the pattern expands to nothing and the loop body runs **0 times**.

**7.** By default `*` skips hidden (dot) files. Show that `echo *` omits `.env`/`.hidden.log`, then enable `shopt -s dotglob` and show `echo *` now includes them.
*Acceptance:* after `dotglob`, `.env` and `.hidden.log` appear in the `echo *` output.

**8.** Enable extended globs with `shopt -s extglob`, then:
- list files that are either `.csv` **or** `.txt` using `*.@(csv|txt)`;
- print every file **except** the `.log` ones using `!(*.log)`.
*Acceptance:* `*.@(csv|txt)` ŌåÆ `data.csv data.txt`; `!(*.log)` ŌåÆ all names except the `.log` files.
*ŌÜĀ’ĖÅ Note:* `extglob` must be enabled **before** the shell **reads** these extended patterns ŌĆö so in an interactive shell run `shopt -s extglob` on its own first, or put it at the **top** of the script. Written as a one-liner, `bash -c 'shopt -s extglob; ls *.@(csv|txt)'` **fails with a syntax error** (the whole line is parsed at once, before extglob takes effect).
