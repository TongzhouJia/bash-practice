# KP B2.7 — Field & Set Tools
**`cut` · `sort` (`-n -r -k -t`) · `uniq -c/-d/-u` · `tr` (`-d -s`) · `paste` · `join` · `column -t`**

**10 tasks · ⭐⭐ · RHEL 10 · server1 · run each against the sample files and match the result**

> Samples are in `samples/`. These are awk's **lightweight companions**: pick columns (`cut`), sort (`sort`), dedup and count (`uniq`), translate characters (`tr`), join (`paste`/`join`), align (`column`). For simple cases they are quicker to reach for than awk.
> ⚠️ `uniq` **only collapses adjacent duplicates**, so you almost always `sort` first.

---

**1.** Extract usernames (field 1) from `passwd.sample` with `cut`.
*Acceptance:* `cut -d: -f1` → `root daemon bin tongzhou …`.

**2.** Extract user and shell (fields 1 and 7).
*Acceptance:* `cut -d: -f1,7` → `root:/bin/bash`, `daemon:/usr/sbin/nologin`, …

**3.** Sort `passwd.sample` **numerically by UID** (field 3).
*Acceptance:* `sort -t: -k3 -n` → order `root`(0), `daemon`(1), `bin`(2), `tongzhou`(1000), …

**4.** Tally HTTP status codes from `access.log` using the `sort | uniq -c` idiom.
*Acceptance:* `awk '{print $9}' … | sort | uniq -c` → `17 200`, `2 301`, `1 401`, `5 404`, `3 500`, `2 503`.

**5.** Find the single busiest IP with the same idiom.
*Acceptance:* `awk '{print $1}' … | sort | uniq -c | sort -rn | head -1` → `10 10.0.0.1`.

**6.** Uppercase a string with `tr`.
*Acceptance:* `echo "hello world" | tr 'a-z' 'A-Z'` → `HELLO WORLD`.

**7.** Two `tr` cleanups: (a) delete all digits from `a1b2c3`; (b) squeeze runs of spaces in `a   b     c` down to one.
*Acceptance:* `tr -d '0-9'` → `abc`; `tr -s ' '` → `a b c`.

**8.** Use `paste` to put each username next to its shell (tab-separated), by combining two `cut` streams.
*Acceptance:* `paste <(cut -d: -f1 passwd.sample) <(cut -d: -f7 passwd.sample)` → `root⇥/bin/bash`, …

**9.** Use `join` to merge "uid → name" and "uid → shell" into `uid name shell` (join on UID).
*Acceptance:* `join <(awk -F: '{print $3,$1}' passwd.sample | sort) <(awk -F: '{print $3,$7}' passwd.sample | sort)` → `0 root /bin/bash`, `1 daemon …`, `1000 tongzhou /bin/bash`, …

**10.** Align the status tally into neat columns with `column -t`.
*Acceptance:* `awk '{print $9}' … | sort | uniq -c | column -t` → two aligned columns (`17  200`, `2   301`, …).
