# KP B4.3 — Process & Service Monitoring
**`pgrep`/`pkill` · `ps --sort` · `systemctl is-active`/`is-enabled` · watchdogs · service check scripts**

**10 tasks · ⭐⭐ · RHEL 10 · server1 · run each and match the result**

> Monitoring whether a process is still there and a service is still up, and restarting it if not. ⚠️ When practicing `kill`/`pkill`, start your own `sleep 300 &` as a target — do not kill real processes.
> Watch the service names: this example uses `cron`, but **on RHEL it is `crond`**; SSH is `sshd`.

---

**1.** Is a process named `sleep` running? (start `sleep 300 &` first).
*Acceptance:* `pgrep -x sleep >/dev/null && echo yes` → `yes`.

**2.** Print the PID(s) of `sleep`.
*Acceptance:* `pgrep -x sleep` → one PID per line.

**3.** Count how many `sleep` processes run.
*Acceptance:* `pgrep -xc sleep` → e.g. `2`.

**4.** Is the `cron` service active?
*Acceptance:* `systemctl is-active cron` → `active` (on RHEL use `crond`).

**5.** Is `cron` enabled at boot?
*Acceptance:* `systemctl is-enabled cron` → `enabled`.

**6.** Watchdog logic: if a service is **not** active, print what you'd do to restart it.
*Acceptance:* `if systemctl is-active --quiet cron; then echo up; else echo "DOWN -> systemctl restart cron"; fi`.

**7.** PID and name of the top-CPU process.
*Acceptance:* `ps -eo pid,comm,%cpu --sort=-%cpu --no-headers | head -1`.

**8.** Kill one specific process by PID (kill the `sleep` you started) and confirm it's gone.
*Acceptance:* `kill <pid>` then `pgrep` no longer lists it.

**9.** Kill all `sleep` processes by name with `pkill`.
*Acceptance:* `pkill -x sleep` → afterwards `pgrep -xc sleep` is `0`.

**10.** Write `svcmon.sh <svc>...` that checks each service, prints `OK`/`DOWN`, and exits non-zero if any is down.
*Acceptance:* `./svcmon.sh cron nosuchsvc` → `OK cron active`, `DOWN nosuchsvc (state: inactive)`, exit `1`.
